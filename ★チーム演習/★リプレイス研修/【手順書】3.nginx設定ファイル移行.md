# nginx バージョンアップ・AP分離・HTTPS化 手順書

---

# 概要

本手順では、既存環境の nginx 構成を改善するために以下の変更を行う。

1. nginx のバージョンアップ（1.16 → 1.28）
2. nginx と AP（Tomcat）のサーバ分離
3. HTTP → HTTPS の強制リダイレクト
4. nginx 設定のモジュール化（conf.d管理）

これにより、セキュリティ・可用性・運用性を向上させる。

---

# 9 検証環境証明書

## 理由

Let's Encrypt未導入

---

## 対応

自己署名証明書

---

## 作成コマンド

openssl req -x509 -nodes -days 365 
-newkey rsa:2048 
-keyout nginx.key 
-out nginx.crt

---

## コマンドの意味

| オプション | 意味 |
|---|---|
| -x509 | 自己署名 |
| -nodes | パス無し |
| -days | 有効期限 |
