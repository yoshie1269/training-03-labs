## 2 nginx設定（Reverse Proxy設定）

### 編集ファイル
```bash
vi /etc/nginx/nginx.conf
```

### 追記場所
```bash
server {
listen 80;
}
```
の **serverブロックの中に追記する**


### 追記内容

下記を **server{} の中に追加**

```bash
server_name  www2.teame2.entrycl.net;  
index index.html;

location / {

proxy_pass http://tomcat_backend;

proxy_http_version 1.1;  
proxy_set_header Connection "";

proxy_set_header Host              $host;  
proxy_set_header X-Real-IP         $remote_addr;  
proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;  
proxy_set_header X-Forwarded-Proto $scheme;

proxy_connect_timeout 5s;  
proxy_read_timeout    60s;  
proxy_send_timeout    60s;

}
```


### server{} の完成イメージ
```bash
server {

listen       80;  
listen       [::]:80;  
server_name  www2.teame2.entrycl.net;  
root         /usr/share/nginx/html;  
index index.html;

include /etc/nginx/default.d/*.conf;

error_page 404 /404.html;

location = /404.html {
}

error_page 500 502 503 504 /50x.html;

location = /50x.html {
}

location / {

proxy_pass http://tomcat_backend;

proxy_http_version 1.1;  
proxy_set_header Connection "";

proxy_set_header Host              $host;  
proxy_set_header X-Real-IP         $remote_addr;  
proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;  
proxy_set_header X-Forwarded-Proto $scheme;

proxy_connect_timeout 5s;  
proxy_read_timeout    60s;  
proxy_send_timeout    60s;

}

}
```

## 5 Tomcat upstream設定

### 編集ファイル
```bash
vi /etc/nginx/conf.d/tomcat.conf
```

### 追記内容（ファイルに新規作成）
```bash
upstream tomcat_backend {

server 172.31.29.127:8080;  
keepalive 32;

}

server {

listen 80;  
server_name _;

location = /health {

return 200 "ok\n";  
add_header Content-Type text/plain;

}

}
```

# SSL証明書更新（cronで実施）

Let's Encrypt証明書は通常 **90日有効** であり  
certbot により自動更新される。

ただし手動更新を行う場合は  
**期限が近い証明書のみ更新対象**となるため  
強制更新を行うことがある。

---

## 1 現在の証明書期限を確認
```bash
openssl x509 -in /etc/letsencrypt/live/www2.teame2.entrycl.net/fullchain.pem -noout -dates
```

## 確認例
```bash
notBefore=Mar  5 00:00:00 2026 GMT  
notAfter=Jun  3 00:00:00 2026 GMT  
```
---

## 2 証明書強制更新
```bash
certbot renew --force-renewal
```

### 注意

通常の更新

certbot renew

では

**有効期限が近い証明書のみ更新**

となるため  
**強制更新オプション**を使用する。

---

## 3 nginxへ証明書反映
```bash
systemctl reload nginx
```

---

## 4 更新後の期限確認
```bash
openssl x509 -in /etc/letsencrypt/live/www2.teame2.entrycl.net/fullchain.pem -noout -dates
```
---

### 成功確認

notAfter の日付が更新されていれば成功
