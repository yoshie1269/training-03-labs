# Let's Encrypt 証明書発行 + nginx HTTPS化 手順書

---

# 概要

本手順では Webサーバ（nginx）に Let's Encrypt のSSL証明書を導入し、  
HTTPS通信を有効化する。

また nginx を Reverse Proxy として動作させ、  
HTTPS通信を受けたリクエストを APサーバ（Tomcat）へ転送する。

最終的な通信経路
```bash
Client
↓
HTTPS :443
↓
nginx(Web Server)
↓ proxy_pass
HTTP :8080
↓
Tomcat(AP Server)
```
---

# 1 nginxが80番ポートで待ち受けているか確認

## この工程でしていること

Let's Encrypt は **HTTPによるドメイン認証（ACME Challenge）** を行う。

そのため nginx が **80番ポートで待ち受けている必要がある**。

もし80番が閉じている場合、証明書発行は失敗する。

---

## 実行コマンド
```bash
ss -tulpn | grep :80
```
---

## コマンドの意味

### ss
ソケット（ポート）を確認するコマンド

### -t
TCP通信を表示

### -u
UDP通信を表示

### -l
LISTEN状態のみ表示

### -p
どのプロセスがポートを使っているか表示

### -n
ポート番号を数値表示

---

## 確認結果例
```bash
tcp LISTEN 0 511 0.0.0.0:80
tcp LISTEN 0 511 [::]:80
```
---

## OKの目安

- nginx が 80 番ポートで LISTEN している
- HTTPアクセスが可能

# 6 nginx HTTPS + Reverse Proxy設定

## nginx.conf
```bash
/etc/nginx/nginx.conf
```
---

## 設定内容
```bash
server {
server_name  www2.teame2.entrycl.net;
index index.html;
root /usr/share/nginx/html;

listen [::]:443 ssl ipv6only=on;
listen 443 ssl;

ssl_certificate
/etc/letsencrypt/live/www2.teame2.entrycl.net/fullchain.pem;

ssl_certificate_key
/etc/letsencrypt/live/www2.teame2.entrycl.net/privkey.pem;

include /etc/letsencrypt/options-ssl-nginx.conf;
ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

location / {

    proxy_pass http://tomcat_backend;

    proxy_http_version 1.1;
    proxy_set_header Connection "";

    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    proxy_connect_timeout 5s;
    proxy_read_timeout 60s;
    proxy_send_timeout 60s;
}

}
```
---

## proxy_pass の意味

proxy_pass http://tomcat_backend;

HTTPSで受けたリクエストを

Tomcat(APサーバ)

へ転送する。
