# NTP構築手順（chrony）

本手順では **NTPサーバとNTPクライアントを構築し、サーバ間で時刻同期を行う**。  
Amazon Linux 2023 では **chrony（chronyd）** が標準のNTP実装として使用される。

---

## 構成イメージ
```bash
NTP Client
↓
NTP Server (chrony)
↓
AWS NTP
169.254.169.123
```
---

## 前提

|項目|内容|
|---|---|
OS|Amazon Linux 2023|
NTPサーバ例|172.31.22.198|
NTPクライアント|172.31.x.x|
AWS NTP|169.254.169.123|

## セキュリティグループ設定（重要）

### この工程でしていること

NTPは **UDP 123ポート** を使用する。  
そのため NTPサーバのセキュリティグループで **クライアントからのNTP通信を許可**する必要がある。



## 追加ルール

### インバウンドルール

|タイプ|プロトコル|ポート|送信元|
|---|---|---|---|
NTP|UDP|123|172.31.0.0/16|

---

## 意味

- UDP 123 は **NTP通信ポート**
- クライアントからNTPサーバへの問い合わせを許可

# ① NTPサーバ構築

## 1 タイムゾーン設定

### この工程でしていること

サーバのタイムゾーンを **日本時間（JST）** に変更する。

### 現在確認
```bash
timedatectl
```


### 変更
```bash
timedatectl set-timezone Asia/Tokyo
```

### 確認
```bash
timedatectl
```

### 確認結果例
```bash
Time zone: Asia/Tokyo (JST)
```
---

## 2 chronyインストール

### この工程でしていること

NTPサービス **chrony** をインストールする。

## インストール
```bash
dnf install -y chrony
```

### インストール確認
```bash
rpm -qa | grep chrony
```

---

# 3 chrony設定ファイル編集

### 編集ファイル
```bash
/etc/chrony.conf
```

### バックアップ作成
```bash
cp -a /etc/chrony.conf /etc/chrony.conf.bak_$(date +%Y%m%d_%H%M%S)
```

### 確認
```bash
ls /etc/ | grep chrony
```


### 編集
```bash
vi /etc/chrony.conf
```

### 追記する内容
```bash
allow 172.31.0.0/16
```
```bash
server 169.254.169.123 prefer iburst
```

### 意味

AWSが提供する **内部NTPサーバ** を使用する。

|オプション|意味|
|---|---|
prefer|優先同期サーバ|
iburst|起動時に高速同期|


### コメントアウトを外す
```bash
local stratum 10
```

### 意味

外部NTPが使えない場合に  
**自サーバを時刻源として動作させる**

---

## 4 chrony起動

### 起動
```bash
systemctl start chronyd
```

### 自動起動
```bash
systemctl enable chronyd
```bash


## 5 状態確認

### サービス状態
```bash
systemctl status chronyd
```

### 同期状況確認
```bash
chronyc sources -v
```

### 時刻確認
```bash
date
```


### クライアント接続確認

クライアント設定後
```bash
chronyc clients
```

# ② NTPクライアント構築

## 1 chronyインストール

### インストール
```bash
dnf install -y chrony
```

### 確認
```bash
rpm -qa | grep chrony
```

---

## 2 chrony設定ファイル編集

### 編集ファイル
```bash
/etc/chrony.conf
```

### バックアップ
```bash
cp -a /etc/chrony.conf /etc/chrony.conf.bak_$(date +%Y%m%d_%H%M%S)
```

### 確認
```bash
ls /etc/ | grep chrony
```

### 編集
```bash
vi /etc/chrony.conf
```

### コメントアウト

#sourcedir /run/chrony.d



### 追記

NTPサーバのIPアドレス
```bash
server 172.31.22.198 iburst
```


### 設定の意味

|設定|意味|
|---|---|
server|同期先NTPサーバ|
172.31.22.198|NTPサーバIP|
iburst|起動時高速同期|

---

## 3 chrony起動


### 起動
```bash
systemctl start chronyd
```


### 自動起動
```bash
systemctl enable chronyd
```

### サービス確認

### 時刻確認
```bash
date
```

### タイムゾーン変更（必要な場合）

### 正常動作の目安

NTPクライアントで

```bash
chronyc sources -v
```

を実行したとき
```bash
^* NTP_SERVER
```
のように **"*" が表示されれば同期成功**
