# Let's Encrypt 証明書発行 + nginx HTTPS化 手順書（Tomcat Reverse Proxy構成）

---

# 概要

本手順では Webサーバ（nginx）で **Let's Encrypt のSSL証明書を取得し、HTTPS通信を有効化**する。

また nginx を **Reverse Proxy** として利用し、  
HTTPSで受けた通信を APサーバ（Tomcat）へ転送する構成を構築する。

---

# 3 Tomcat upstream設定

## この工程でしていること

nginxが Tomcatへ通信を転送できるように  
**Reverse Proxy設定**を作成する。

---

## 設定ファイル
```bash
/etc/nginx/conf.d/tomcat.conf
```
---

## 設定内容
```bash
upstream tomcat_backend {

server 172.31.29.127:8080;
keepalive 32;

}

server {

listen 80;
server_name www2.teame2.entrycl.net;

location = /health {
    return 200 "ok\n";
    add_header Content-Type text/plain;
}

location / {

    proxy_pass http://tomcat_backend;

    proxy_http_version 1.1;
    proxy_set_header Connection "";

    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    proxy_connect_timeout 5s;
    proxy_read_timeout 60s;
    proxy_send_timeout 60s;
}

}
```

---

# upstream 設定の意味

### upstream tomcat_backend

Tomcatサーバのグループを定義

tomcat_backend

という名前で呼び出せる。

---

### server 172.31.29.127:8080

Tomcatアプリケーションサーバ
```bash
IP : 172.31.29.127
Port : 8080
```
---

### keepalive

接続再利用

性能向上のための設定。

---

# proxy設定の意味
```bash
proxy_pass http://tomcat_backend;
```
nginxが受けたHTTP/HTTPS通信を  
Tomcatへ転送する。

---

### X-Forwarded系ヘッダ

Tomcatにクライアント情報を伝える

|ヘッダ|意味|
|---|---|
X-Real-IP|クライアントIP|
X-Forwarded-For|プロキシ経由IP|
X-Forwarded-Proto|HTTP / HTTPS|

# 8 nginx HTTPS設定

Certbotにより  
HTTPS server 設定が自動追加される。

---

## 設定ファイル
```bash
/etc/nginx/conf.d/tomcat.conf
```
---

## 追加される設定
```bash
server {

listen 443 ssl;
server_name www2.teame2.entrycl.net;

ssl_certificate
/etc/letsencrypt/live/www2.teame2.entrycl.net/fullchain.pem;

ssl_certificate_key
/etc/letsencrypt/live/www2.teame2.entrycl.net/privkey.pem;

include /etc/letsencrypt/options-ssl-nginx.conf;

location / {

    proxy_pass http://tomcat_backend;

}

}
```
