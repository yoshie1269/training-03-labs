# Java (Amazon Corretto) パッケージ確認

## 実施コマンド
```bash
dnf list | grep corretto
```
## この作業でしていること
Amazon Linux 2023 のリポジトリに存在する **Java (Amazon Corretto) のバージョン一覧を確認**している。  
どのバージョンのJDKがインストール可能かを把握するための調査。

## 確認できる内容
・Java8  
・Java11  
・Java17（LTS）  
・Java21（LTS）  
・Java22〜25（最新系）

## 今回選択したバージョン
java-17-amazon-corretto

理由  
・LTS（長期サポート）  
・Tomcat9と互換性が高い  
・実運用でもよく使われる安定版


# Java17 (Amazon Corretto) インストール

## 実施コマンド
```bash
sudo dnf install java-17-amazon-corretto -y
```
## この作業でしていること
Amazon Corretto 17 (OpenJDK17) をサーバにインストールしている。  
Tomcatを動作させるために **Java実行環境 (JRE/JDK)** を準備する作業。

## 同時にインストールされた主な依存パッケージ
・java-17-amazon-corretto-headless  
・fontconfig  
・freetype  
・libX11  
・cairo  
・alsa-lib

※ JavaのGUI機能やフォント処理に必要なライブラリ

# Tomcatユーザー作成

## 実施コマンド
```bash
sudo useradd -r -m -U -d /usr/local/tomcat -s /bin/false tomcat
```

## この作業でしていること
Tomcat専用の実行ユーザーを作成している。

## オプション説明
```bash
-r  
システムユーザーとして作成

-m  
ホームディレクトリを作成

-U  
同名のグループ作成

-d /usr/local/tomcat  
ホームディレクトリ指定

-s /bin/false  
ログイン不可ユーザー
```

## 目的
Tomcatを **rootではなく専用ユーザーで安全に動作させるため**


# Tomcatダウンロード

## 作業ディレクトリ移動
```bash
cd /tmp
```

## 実施コマンド
```bash
wget https://archive.apache.org/dist/tomcat/tomcat-9/v9.0.12/bin/apache-tomcat-9.0.12.tar.gz
```

## この作業でしていること
Apache公式アーカイブから **Tomcat9.0.12 をダウンロード**している。

## 保存されたファイル
/tmp/apache-tomcat-9.0.12.tar.gz


# Tomcat展開

## 実施コマンド
```bash
sudo tar -xvzf apache-tomcat-9.0.12.tar.gz -C /usr/local/
```

## コマンドの意味
```bash
tar  
アーカイブ操作

-x  
展開

-v  
詳細表示

-z  
gzip解凍

-f  
ファイル指定

-C /usr/local/  
展開先ディレクトリ指定
```

## この作業でしていること
ダウンロードしたTomcatを **/usr/local に展開してインストール**している。


# 展開後のディレクトリ構成（主要）
```bash
/usr/local/apache-tomcat-9.0.12

conf/  
Tomcat設定ファイル

bin/  
起動スクリプト

lib/  
Tomcatライブラリ

logs/  
ログ

temp/  
一時ファイル

webapps/  
アプリケーション配置ディレクトリ
```

# Tomcat起動スクリプト

## 主なスクリプト
```bash
bin/startup.sh  
Tomcat起動

bin/shutdown.sh  
Tomcat停止

bin/catalina.sh  
Tomcat制御
```
