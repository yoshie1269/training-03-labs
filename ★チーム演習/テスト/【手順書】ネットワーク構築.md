# 踏み台接続構築 手順書（鍵切替＋鍵分離方式 / teami）

---

## 1. 目的

本手順書は、web-1 を踏み台サーバとして利用し、
外部接続用鍵と内部接続用鍵を分離した
安全なSSH接続環境を構築することを目的とする。

---

## 2. 使用鍵

| 用途 | 鍵名 | 保管場所 |
|------|------|----------|
| 外部接続 | entrycl_202601.pem | 各自PC |
| 内部接続 | teami_key.pem | web-1 |

---

## 3. 構成概要
```bash
自分PC
　↓
entrycl
　↓
web-1
　↓
teami
　↓
ap / db
```

---

## 4. 担当分担

| 区分 | サーバ | 担当 |
|------|--------|------|
| 踏み台管理 | web-1 | Web① |
| 予備踏み台 | web-2 | Web② |
| AP管理 | ap-1 | AP① |
| AP管理 | ap-2 | AP② |
| DB管理 | db | DB① |

---

# ===============================
# 5. Web-1担当：鍵切替（onishi → entrycl）
# ===============================

【作業場所】自分PC → web-1

---

## 5-1. onishi鍵でログイン
```bash
ssh -i ~/.ssh/onishi.entrycl.pem ec2-user@＜web-1 EIP＞
```
---

## 5-2. entrycl公開鍵取得（PC）
```bash
ssh-keygen -y -f ~/.ssh/entrycl_202601.pem
```
→ 出力された文字列をコピー

---

## 5-3. 公開鍵登録（削除禁止）
```bash
vi ~/.ssh/authorized_keys
```
→ 既存行の下に貼り付け

---

## 5-4. 権限確認
```bash
chmod 700 ~/.ssh  
```
```bash
chmod 600 ~/.ssh/authorized_keys
```
---

## 5-5. entryclで接続確認
```bash
exit
```
```bash
ssh -i ~/.ssh/entrycl_202601.pem ec2-user@＜web-1 EIP＞
```
---

## 5-6. onishi鍵削除（確認後のみ）
```bash
vi ~/.ssh/authorized_keys
```
→ onishi行を削除

---

# ===============================
# 6. Web-1担当：内部用鍵作成（teami）
# ===============================

【作業場所】web-1

---

## 6-1. 鍵作成
```bash
ssh-keygen -t rsa -b 4096 -f ~/.ssh/teami_key
```
※ パスフレーズなし

---

## 6-2. 確認
```bash
ls ~/.ssh/teami_key*
```
---

# ===============================
# 7. AP / DB担当：内部鍵登録
# ===============================

【作業場所】web-1 → 各サーバ

---

## 7-1. 接続（仮）
```bash
ssh ec2-user@ap-1.wp.local
```
---

## 7-2. 公開鍵登録
```bash
mkdir -p ~/.ssh  
```
```bash
chmod 700 ~/.ssh  
```
```bash
vi ~/.ssh/authorized_keys
```
→ teami_key.pub を貼付

---

## 7-3. 権限設定
```bash
chmod 600 ~/.ssh/authorized_keys
```
---

## 7-4. 確認
```bash
cat ~/.ssh/authorized_keys
```
---

※ ap-2 / db も同様

---

# ===============================
# 8. Web-1担当：踏み台接続確認
# ===============================

【作業場所】web-1

---
```bash
ssh -i ~/.ssh/teami_key ec2-user@ap-1.wp.local  
```
```bash
ssh -i ~/.ssh/teami_key ec2-user@db.wp.local
```

# 9. Web-2担当：予備踏み台対応（任意）

- teami_key配置
- 接続確認

## 11. 完了条件

以下を満たした場合、本構築は完了とする。

- entrycl鍵でのみweb接続可能
- teami鍵でのみ内部接続可能
- Private直アクセス不可
