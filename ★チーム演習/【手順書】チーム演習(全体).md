# 構築手順書  
（WordPressマネージド構築演習 / teami）

---

## 1. 目的

本手順書は、WordPressマネージド環境における
各サーバの構築作業を標準化し、
引継ぎ可能な状態とすることを目的とする。

---

## 2. 前提条件

- OS：Amazon Linux 2023
- サーバ構成：5台
- 内部DNS使用（/etc/hosts 不使用）
- hostnamectl 未使用
- /etc/resolv.conf 未変更
- すべてホスト名通信とする

# 5. 内部DNS構築（ap-1）

## 5-1. BINDインストール
```bash
dnf install bind bind-utils -y
```
```bash
systemctl enable named
```
```bash
systemctl start named
```
---

## 5-2. named.conf設定
```bash
vi /etc/named.conf
```
```bash
・listen-on：Private IP
・allow-query：VPC範囲
・forwarders：設定しない
```
---

## 5-3. ゾーン定義
```bash
vi /etc/named.conf
```
```bash
zone "wp.local" IN {
  type master;
  file "wp.local.zone";
  allow-transfer { ap-2のIP; };
};
```
---

## 5-4. ゾーンファイル作成
```bash
vi /var/named/wp.local.zone
```
```bash
$TTL 3600
@ IN SOA ap-1.wp.local. root.wp.local. (
  20260201
  3600
  900
  604800
  86400 )

@ IN NS ap-1.wp.local.
@ IN NS ap-2.wp.local.

web-1 IN A xxx.xxx.xxx.xxx
web-2 IN A xxx.xxx.xxx.xxx
ap-1  IN A xxx.xxx.xxx.xxx
ap-2  IN A xxx.xxx.xxx.xxx
db-1  IN A xxx.xxx.xxx.xxx
```
---

## 5-5. 設定チェック
```bash
named-checkconf
```
```bash
named-checkzone wp.local /var/named/wp.local.zone
```
```bash
systemctl restart named
```
---

## 5-6. 接続確認
```bash
dig @ap-1.wp.local web-1.wp.local
```
```bash
dig @ap-1.wp.local db-1.wp.local
```
→ IPが返ること

# 6. 内部DNSスレーブ構築（ap-2）

## 6-1. BIND導入
```bash
dnf install bind bind-utils -y
```
```bash
systemctl enable named
```
```bash
systemctl start named
```
---

## 6-2. スレーブ設定
```bash
vi /etc/named.rfc1912.zones
```
```bash
zone "wp.local" IN {
  type slave;
  masters { ap-1のIP; };
  file "slaves/wp.local.zone";
};
```
---

## 6-3. 再起動
```bash
systemctl restart named
```
---

## 6-4. ゾーン転送確認
```bash
ls /var/named/slaves/
```
→ wp.local.zone が存在
```bash
dig @ap-2.wp.local web-1.wp.local
```

# 10. NFS構築（db-1）
```bash
dnf install nfs-utils -y
```
```bash
mkdir /share/mail
mkdir /share/www
```
```bash
vi /etc/exports
```
```bash
/share/mail *(rw,sync,no_root_squash)
/share/www  *(rw,sync,no_root_squash)
```
```bash
systemctl enable nfs-server
systemctl start nfs-server
```
```bash
exportfs -v
```
---

## クライアント側
```bash

mount -t nfs db-1.wp.local:/share/www /mnt/www
```
