# 踏み台経由SSH 接続手順書  
（Private Subnet サーバ接続 / teami）

---

## 1. 目的

本手順書は、Public Subnet 上の Web-1 を踏み台サーバとして利用し、  
Private Subnet 上の AP / DB サーバへ安全に SSH 接続するための手順を示す。

---

## 2. 前提条件

- 踏み台サーバ：Web-1（Public Subnet）
- 接続先サーバ：AP-1 / AP-2 / DB-1（Private Subnet）
- 使用鍵：learnCL_10_key.pem
- 内部DNSが設定済み（wp.local）
- /etc/hosts は使用しない

---

## 3. 構成イメージ

自分PC → Web-1（踏み台） → Privateサーバ
```bash
PC
↓
Web-1（Public）
↓
AP / DB（Private）
```

---

## 4. 使用ホスト名

| 役割 | ホスト名 |
|------|----------|
| 踏み台 | web-1.wp.local |
| AP-1 | ap-1.wp.local |
| AP-2 | ap-2.wp.local |
| DB-1 | db-1.wp.local |

※ Web-1 は EIP で接続する

---

## 5. 手順①：自分PCから踏み台へ接続

### 5-1. SSH接続

自分PCで以下を実行する。
```bash
ssh -i learnCL_10_key.pem ec2-user@<Web-1のEIP>
```

---

### 5-2. 接続確認

以下が表示されれば成功。

接続できない場合は以下を確認する。

- セキュリティグループの22番許可
- EIPの指定ミス
- 鍵ファイルの指定ミス

---

## 6. 手順②：踏み台に秘密鍵を配置

### 6-1. .ssh ディレクトリ作成

踏み台（Web-1）で実行。
```bash
mkdir -p ~/.ssh
```
```bash
chmod 700 ~/.ssh
```

---

### 6-2. 鍵ファイル転送（推奨）

自分PCで実行。
```bash
scp -i learnCL_10_key.pem 
```
```bash
learnCL_10_key.pem 
```
```bash
ec2-user@<Web-1のEIP>:~/.ssh/
```
