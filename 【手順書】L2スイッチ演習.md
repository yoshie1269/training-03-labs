# ■ L2スイッチ-1  
2台PC間通信（MAC学習・ARP確認）

---

## 【目的】

・L2スイッチの基本動作理解  
・MACアドレス学習の確認  
・ARPの流れを理解する  

---

## 【構成】

PC1 ── SW ── PC2

---

## ① トポロジ作成

### この工程でしていること
・物理的に同一ネットワークを構築  
・L2スイッチ経由でPC同士を接続  

### 手順
・PC ×2 を配置  
・2960 スイッチ ×1 を配置  
・PC1 → Fa0/1  
・PC2 → Fa0/2  
・Copper Straight-Through ケーブル使用  

### なぜ Straight-Through？
異なる機器（PCとSW）間接続のため

---

## ② IPアドレス設定

### この工程でしていること
・同一ネットワーク内に配置  
・ルーターなしで通信可能にする

### 設定値
```bash
PC1  
IP：192.168.1.1  
Subnet：255.255.255.0  
GW：空白  

PC2  
IP：192.168.1.2  
Subnet：255.255.255.0  
GW：空白  
```
### 数字の意味
・192.168.1.0/24 = 同一ネットワーク  
・255.255.255.0 = ホスト部8ビット  

---

## ③ スイッチ初期確認

### 実行コマンド
```bash
enable
```
```bash
show mac address-table
```

### コマンドの意味
| コマンド | 意味 |
|-----------|------|
| enable | 特権モード |
| show mac address-table | MAC学習テーブル表示 |

### OK目安
・何も表示されない（まだ通信していないため）

---

## ④ Ping 実行

PC1で
```bash
ping 192.168.1.2
```
### 何が起きているか
1. ARP要求（ブロードキャスト）
2. PC2がARP応答
3. スイッチが送信元MACを学習
4. ICMP Echo送信

※最初の1回はARPで遅い

---

## ⑤ 再確認
```bash
show mac address-table
```

### OK目安
・Fa0/1 に PC1 のMAC
・Fa0/2 に PC2 のMAC
が表示される

---

## 理解ポイント

・スイッチは「送信元MAC」を学習  
・ARPはIP→MAC変換  
・同一セグメントならルーター不要  

---

# ■ L2スイッチ-2  
VLAN分割

---

## 【目的】

・VLAN作成  
・ブロードキャスト分離確認  
・VLAN間通信不可確認  

---

## 【構成】

VLAN10：PC1, PC2  
VLAN20：PC3, PC4  

---

## ① IP設定
```bash
VLAN10  
PC1 192.168.10.1/24  
PC2 192.168.10.2/24  

VLAN20  
PC3 192.168.20.1/24  
PC4 192.168.20.2/24  

GWは空白
```
### 数字の意味
192.168.10.0/24 と 192.168.20.0/24 は別ネットワーク

---

## ② VLAN作成
```bash
enable
```
```bash
configure terminal
```
```bash
vlan 10
```
```bash
exit
```
```bash
vlan 20
```
```bash
exit
```

### コマンドの意味
| コマンド | 意味 |
|-----------|------|
| configure terminal | 設定モード |
| vlan 10 | VLAN作成 |
| exit | 上位モードへ戻る |

---

## ③ ポート割当
VLAN 10
```bash
interface range fastEthernet 0/1-2
```
```bash
switchport mode access
```
```bash
switchport access vlan 10
```
```bash
exit
```

VLAN 20
```bash
interface range fastEthernet 0/3-4
```
```bash
switchport mode access
```
```bash
switchport access vlan 20
```
```bash
exit
```

### コマンドの意味

| コマンド | 意味 |
|-----------|------|
| interface range | 複数ポート指定 |
| switchport mode access | アクセスポート設定 |
| switchport access vlan | VLAN割当 |

---

## ④ 確認
```bash
show vlan brief
```
### OK目安
・Fa0/1-2 が VLAN10  
・Fa0/3-4 が VLAN20  

---

## ⑤ Ping確認

成功：
PC1 → PC2  
PC3 → PC4  

失敗：
PC1 → PC3  

---

## 理解ポイント

・VLAN = 仮想的なL2ネットワーク  
・同一スイッチでもVLANが違えば通信不可  
・VLAN間通信にはL3（ルーター）が必要  

---

# ■ L2スイッチ-3  
STP（ループ防止）

---

## 【目的】

・ループ構成確認  
・Blockingポート確認  
・ルートブリッジ変更  

---

## 【構成】

SW1 ─ SW2  
SW2 ─ SW3  
SW3 ─ SW1  

三角形構成

---

## ① 接続

SW1 Fa0/1 ↔ SW2 Fa0/1  
SW2 Fa0/2 ↔ SW3 Fa0/1  
SW3 Fa0/2 ↔ SW1 Fa0/2  

---

## ② STP確認
```bash
enable
```
```bash
show spanning-tree
```

### コマンドの意味
| コマンド | 意味 |
|-----------|------|
| show spanning-tree | STP状態表示 |

---

### 確認ポイント

・This bridge is the root  
・Root Port  
・Blocking Port  

---

### OK目安

・ルートブリッジ1台  
・どこか1ポートがBlocking  

---

## ③ ルート変更
```bash
configure terminal
```
```bash
spanning-tree vlan 1 priority 4096
```
```bash
end
```

### 数字の意味
・デフォルト優先度：32768  
・4096 は小さい → ルートになりやすい  
・数値は 4096 の倍数

確実に変更：
```bash
spanning-tree vlan 1 root primary
```

---

## ④ 再確認
```bash
show spanning-tree
```

### OK目安

・指定SWがRootになる  
・Rootは全ポートForwarding  
・他SWでBlockingが発生  

---

## 理解ポイント

・STPはループ防止  
・優先度が小さいほど有利  
・ルート以外にBlockingが発生  
・Blockingがないとブロードキャストストーム発生
