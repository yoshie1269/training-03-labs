# ルータ演習 手順書（ルータ1〜4：基礎ルーティング / 複数PC / HSRP / Static優先→OSPF自動切替）

---

## 概要

本手順書では、ルータを用いた L3 ルーティングの基本から冗長化、経路自動切替までを段階的に実施する。

- ルータ1：異なるネットワーク間のルーティング（基礎）
- ルータ2：スイッチ併用の複数PC間ルーティング
- ルータ3：HSRP によるデフォルトゲートウェイ冗長化（詳細）
- ルータ4：Static を優先し、使えない時だけ OSPF に自動切替（AD による共存）

## 共通の注意点（重要）

- IP / サブネットマスク / デフォルトGW の整合が取れていないと通信できない
- ルータの IF は初期状態で shutdown のため `no shutdown` が必須
- 初回 ping の 1回目 timeout は ARP 学習による可能性がある（異常とは限らない）
- 段階確認（PC→GW→相手）が最短で原因特定できる


<br>

---

# ■ ルータ1：異なるネットワーク間のルーティング（基礎）

## 目的
- 異なるネットワーク同士は直接通信できないことを理解する
- ルータが L3（IP）で中継する仕組みを理解する
- デフォルトゲートウェイの役割を理解する

---

## 1-1. 構成概要

### この工程でしていること
- PC1 と PC2 を別セグメントに分け、ルータを中継点にする

### 構成
```bash
PC1 –– Router –– PC2

PC1ネットワーク：192.168.1.0/24
PC2ネットワーク：192.168.2.0/24
```

---

## 1-2. IP設計

### この工程でしていること
- PC と ルータIF に IP を割り当て、L3 で通信できる形にする

### 設定値
PC1
```bash
IP：192.168.1.10/24
GW：192.168.1.1

Router g0/0

IP：192.168.1.1/24
```

PC2
```bash
IP：192.168.2.10/24
GW：192.168.2.1

Router g0/1

IP：192.168.2.1/24
```

### ポイント
- 同一ネットワーク内は L2（MAC）で届く
- 別ネットワーク宛は「デフォルトGW」に渡してルータに中継させる

---

## 1-3. ルータ設定

### この工程でしていること
- ルータが両ネットワークの「直結ネットワーク」を持つ状態を作る
- IF を up にしてリンクを有効化する


### 実行コマンド
```bash
enable
conf t

interface g0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit

interface g0/1
ip address 192.168.2.1 255.255.255.0
no shutdown
exit

end
```

### コマンドの意味

| コマンド | 意味 |
|---|---|
| enable | 特権モード |
| conf t | グローバル設定モード |
| interface | IF選択 |
| ip address | IP設定 |
| no shutdown | IF有効化 |
| end | 設定終了 |

---

## 1-4. 確認

### この工程でしていること
- IF の状態と、直結ルート（Connected）が載ったことを確認する

### 確認コマンド
```bash
show ip interface brief
show ip route
```

### OKの目安
- IF が up/up
- ルートに以下がある
```bash
C 192.168.1.0/24
C 192.168.2.0/24

※ C = Connected（直結ネットワーク）
```

---

## 1-5. 通信確認

### 実行内容
PC1 から PC2 へ
```bash
ping 192.168.2.10
```

### OKの目安
- Reply が返る

### 通信の流れ（理解ポイント）
```bash
PC1 → GW(192.168.1.1)
→ ルータが宛先 192.168.2.0/24 をルーティングテーブルで判断
→ g0/1 へ転送
→ PC2へ到達
```

## 学習ポイント
- ルータは IP（L3）で転送
- デフォルトGWがないと別ネットワークへ行けない
- 最初の1回 timeout は ARP 学習の可能性

<br>

---

# ■ ルータ2：スイッチ併用の複数PC間ルーティング

## 目的
- 同一ネットワーク内はスイッチ（L2）で通信できる
- 異なるネットワーク間はルータ（L3）が必要である

---

## 2-1. 構成

### この工程でしていること
- 172.31 側・10.0 側それぞれに PC を複数置き、スイッチで収容する
- 2つのネットワークをルータで中継する


### 構成
```bash
PC02,03 – SW1 – Router – SW2 – PC04,05

ネットワーク1：172.31.0.0/24
ネットワーク2：10.0.0.0/24
```

---

## 2-2. IP設計

### 172.31側
```bash
PC02：172.31.0.2/24  GW 172.31.0.1
PC03：172.31.0.3/24  GW 172.31.0.1
R g0/0：172.31.0.1/24
```
### 10.0側
```bash
PC04：10.0.0.4/24  GW 10.0.0.1
PC05：10.0.0.5/24  GW 10.0.0.1
R g0/1：10.0.0.1/24
```

---

## 2-3. ルータ設定

### 実行コマンド
```bash
enable
conf t

interface g0/0
ip address 172.31.0.1 255.255.255.0
no shutdown
exit

interface g0/1
ip address 10.0.0.1 255.255.255.0
no shutdown
exit

end
```

### この工程でしていること
- 2つの異なるネットワークをルータが保持し、L3 転送可能にする

---

## 2-4. スイッチの役割（理解）

### この工程でしていること
- 同一ネットワーク内の転送（MAC学習）を担当する

### ポイント
- スイッチは L2（MAC）
- 基本 VLAN1 のままで通信できる（特別な設定不要）

---

## 2-5. 確認

### 確認コマンド
```bash
show ip interface brief
show ip route
```

### OKの目安
```bash
C 172.31.0.0/24
C 10.0.0.0/24
```

---

## 2-6. 通信確認（段階的）

### 同一ネットワーク確認
PC02 → PC03
```bash
ping 172.31.0.3
```
PC04 → PC05
```bash
ping 10.0.0.5
```

### 異ネットワーク確認
PC02 → PC04
```bash
ping 10.0.0.4
```

### 通信の流れ

PC02 → GW(172.31.0.1) → Router → 10.0.0.0/24 へ転送 → PC04

## 学習ポイント
- スイッチは L2、ルータは L3
- ネットワークが異なるとルータが必須
- PC→GW→相手の順で確認すると早い

<br>

# ■ ルータ3：HSRP によるデフォルトGW冗長化（詳細）

## 目的
- PC(172.16.0.0/24) と Server(192.168.0.0/24) 間の通信を確立
- デフォルトルータ停止でも通信継続
- VIP / priority / preempt の意味を理解

---

## 3-1. 構成概要

### この工程でしていること
- ルータ2台（R1/R2）で VIP を共有し、GW を仮想化する
- Active が VIP を保持し、障害時に Standby が引き継ぐ

---

## 3-2. IP設計

### PC側（172.16.0.0/24）
```bash
PC06：172.16.0.10/24  GW 172.16.0.254
PC07：172.16.0.11/24  GW 172.16.0.254

R1 g0/0/0：172.16.0.1/24
R2 g0/0/0：172.16.0.2/24
VIP：172.16.0.254
```
### Server側（192.168.0.0/24）
```bash
Server01：192.168.0.10/24  GW 192.168.0.254
Server02：192.168.0.11/24  GW 192.168.0.254

R1 g0/0/1：192.168.0.1/24
R2 g0/0/1：192.168.0.2/24
VIP：192.168.0.254
```

### 重要ポイント
- PC/Server の GW は「実IP」ではなく「VIP」
- 実IPをGWにすると冗長化にならない

---

## 3-3. R1 設定（Activeにする）

### この工程でしていること
- priority を高くして R1 を Active にする
- preempt を有効にして復旧時に R1 が Active に戻るようにする


### 実行コマンド（例：priority 110）
```bash
enable
conf t

interface g0/0/0
ip address 172.16.0.1 255.255.255.0
standby 1 ip 172.16.0.254
standby 1 priority 110
standby 1 preempt
no shutdown
exit

interface g0/0/1
ip address 192.168.0.1 255.255.255.0
standby 2 ip 192.168.0.254
standby 2 priority 110
standby 2 preempt
no shutdown
exit

end
```

---

## 3-4. R2 設定（Standbyにする）

### この工程でしていること
- priority を R1 より低くし、Standbyにする

### 実行コマンド（例：priority 100）
```bash
enable
conf t

interface g0/0/0
ip address 172.16.0.2 255.255.255.0
standby 1 ip 172.16.0.254
standby 1 priority 100
standby 1 preempt
no shutdown
exit

interface g0/0/1
ip address 192.168.0.2 255.255.255.0
standby 2 ip 192.168.0.254
standby 2 priority 100
standby 2 preempt
no shutdown
exit

end
```

### 注意点
- priority 未設定だと 100 同士になり、IPが大きい方が Active になる（意図しないActive）

---

## 3-5. 状態確認

### 確認コマンド
```bash
show ip interface brief
show standby brief
```

### OKの目安
- R1：Grp1 Active / Grp2 Active
- R2：Grp1 Standby / Grp2 Standby

---

## 3-6. 通信確認

### PC→Server
```bash
ping 192.168.0.10
ping 192.168.0.11
```
### Server→PC
```bash
ping 172.16.0.10
ping 172.16.0.11
```

### OKの目安
- 通信成功

---

## 3-7. フェイルオーバー試験

### この工程でしていること
- R1 を停止し、R2 が Active を引き継ぐことを確認する

### 手順
PCで連続 ping（例）
```bash
ping -t 192.168.0.10
```
R1 停止（両IF停止が分かりやすい）
```bash
conf t
interface g0/0/0
shutdown
exit
interface g0/0/1
shutdown
end
```

### OKの目安
- 数回 timeout 後に ping が復帰する
- R2 が Active になる（show standby brief）

### なぜ数回止まるか
- HSRP は Hello/Hold タイマーで切替するため

---

## 3-8. 復旧試験

### 実行コマンド
```bash
conf t
interface g0/0/0
no shutdown
exit
interface g0/0/1
no shutdown
end
```

### OKの目安
- preempt により R1 が Active を奪い返す

---

## 3-9. 重要理解ポイント
- VIP は「ゲートウェイの仮想化」
- priority は「誰を Active にするか」
- preempt は「復旧時に元へ戻す」
- セグメントごとに HSRP は独立動作
- 片側IFだけ落とすと分断が起きることがある（片系だけ死ぬ）


<br>

# ■ ルータ4：Static優先 / 使えない時は OSPF へ自動切替

## 目的
- 通常時：Static（AD=1）で最優先ルーティング
- 障害時：Staticが消えたら OSPF（AD=110）へ自動切替
- 「Staticメイン」「OSPFバックアップ」を AD で実現

---

## 4-1. 構成（三角構成：3ルータ + 両端PC）

### この工程でしていること
- R1-R2 をメイン経路（Static）
- R1-R3-R2 をバックアップ経路（OSPF）
- R1-R2 を落とすと OSPF が採用される


### 構成
```bash
PC01 – R1 – R2 – PC02
        \    /
         \  /
          R3
```

---

## 4-2. IP設計（例）

### LAN-A（PC01側）
```bash
PC01：192.168.1.10/24  GW 192.168.1.1
R1 g0/0：192.168.1.1/24
```

### LAN-B（PC02側）
```bash
PC02：192.168.2.10/24  GW 192.168.2.1
R2 g0/0：192.168.2.1/24
```
### ルータ間リンク（/30：点対点）
```bash
R1-R2：10.0.12.0/30
R1 g0/1：10.0.12.1/30
R2 g0/1：10.0.12.2/30

R1-R3：10.0.13.0/30
R1 g0/2：10.0.13.1/30
R3 g0/0：10.0.13.2/30

R3-R2：10.0.23.0/30
R3 g0/1：10.0.23.1/30
R2 g0/2：10.0.23.2/30
```

---

## 4-3. 初期設定：各IFに IP 付与 + no shutdown

### この工程でしていること
- 各ルータが直結ネットワーク（Connected）を持つ状態を作る

### R1
```bash
enable
conf t
interface g0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit
interface g0/1
ip address 10.0.12.1 255.255.255.252
no shutdown
exit
interface g0/2
ip address 10.0.13.1 255.255.255.252
no shutdown
exit
end
```

### R2
```bash
enable
conf t
interface g0/0
ip address 192.168.2.1 255.255.255.0
no shutdown
exit
interface g0/1
ip address 10.0.12.2 255.255.255.252
no shutdown
exit
interface g0/2
ip address 10.0.23.2 255.255.255.252
no shutdown
exit
end
```

### R3
```bash
enable
conf t
interface g0/0
ip address 10.0.13.2 255.255.255.252
no shutdown
exit
interface g0/1
ip address 10.0.23.1 255.255.255.252
no shutdown
exit
end
```

### 確認方法とOKの目安
```bash
show ip interface brief
show ip route
```
- IF が up/up
- C（Connected）が載る（例：C 10.0.12.0/30 ...）

---

## 4-4. OSPF 設定（バックアップ用：常時動かす）

### この工程でしていること
- 隣接（Neighbor）を作り、代替経路を学習できる土台を作る
- 通常は Static が優先されるため、OSPFは待機になる

### R1
```bash
conf t
router ospf 1
router-id 1.1.1.1
network 10.0.12.0 0.0.0.3 area 0
network 10.0.13.0 0.0.0.3 area 0
network 192.168.1.0 0.0.0.255 area 0
passive-interface g0/0
end
```

### R2
```bash
conf t
router ospf 1
router-id 2.2.2.2
network 10.0.12.0 0.0.0.3 area 0
network 10.0.23.0 0.0.0.3 area 0
network 192.168.2.0 0.0.0.255 area 0
passive-interface g0/0
end
```

### R3
```bash
conf t
router ospf 1
router-id 3.3.3.3
network 10.0.13.0 0.0.0.3 area 0
network 10.0.23.0 0.0.0.3 area 0
end
```

### 確認方法とOKの目安
```bash
show ip ospf neighbor
```
- Neighbor が表示される

---

## 4-5. Static ルート設定（通常経路：最優先）

### この工程でしていること
- 通常時は R1↔R2 のメインリンクで通信させる
- Static(AD=1) が OSPF(AD=110) より優先されるため、Staticが採用される

### 注意点（超重要）
- next-hop は「隣ルータの直結リンク側IP」を指定する
- LAN側IP（192.168.2.1 等）を next-hop に指定すると誤りになりやすい

### R1（LAN-B へは R2 経由固定）
```bash
conf t
ip route 192.168.2.0 255.255.255.0 10.0.12.2
end
```

### R2（LAN-A へは R1 経由固定）
```bash
conf t
ip route 192.168.1.0 255.255.255.0 10.0.12.1
end
```

### 確認方法とOKの目安
R1
```bash
show ip route 192.168.2.0
```
- `S 192.168.2.0/24 [1/0] via 10.0.12.2` が出る

R2
```bash
show ip route 192.168.1.0
```
- `S 192.168.1.0/24 [1/0] via 10.0.12.1` が出る

---

## 4-6. 通信確認（通常時：Static で通る）

### 段階確認
PC01 → R1（GW）
```bash
ping 192.168.1.1
```
PC02 → R2（GW）
```bash
ping 192.168.2.1
```
PC01 → PC02
```bash
ping 192.168.2.10
```

### OKの目安
- ping 成功（通常は Static 経路で通る）

---

## 4-7. 障害試験：Staticが使えない時に OSPF へ切替

### この工程でしていること
- メインリンク（R1-R2）を落とし、Staticが成立しない状態にする
- OSPF が代替経路（R1→R3→R2）を採用することを確認する


### R1 でメインリンク停止（R1-R2）
```bash
conf t
interface g0/1
shutdown
end
```

### 期待する結果（OKの目安）
R1 の経路が OSPF に切り替わる
```bash
show ip route 192.168.2.0
```
例（想定）
```bash
O 192.168.2.0/24 [110/…] via 10.0.13.2
```
※ ping は数回落ちても復帰すればOK（OSPF収束）


### 追加確認
```bash
show ip ospf neighbor
```
- R3 との隣接が残っている


### 復旧（リンクを戻す）
```bash
conf t
interface g0/1
no shutdown
end
```

### 復旧後の期待
- Static（S）が自動で再採用される（AD=1 が最優先）
```bash
show ip route 192.168.2.0
```
- S に戻る

---

## 4-8. 学習ポイントまとめ（ルータ4）

- Static(AD=1) は OSPF(AD=110) より優先される
- Staticが“使えなくなる”と経路が消え、OSPFが採用される（自動切替）
- next-hop 指定は「隣ルータの直結リンクIP」が鉄則
- 確認順は以下が早い
```bash
show ip interface brief → show ip ospf neighbor → show ip route
```
