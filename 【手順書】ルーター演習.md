# ルーター演習 手順書（ルーター① / ② / ③：HSRP）

---

## 概要

本手順書では、ルータを用いた基本的な L3 接続と、HSRP によるデフォルトゲートウェイ冗長化を行う。

- ルーター①：2台PCを別ネットワークに分離し、ルータで疎通させる
- ルーター②：172.31.0.0/24 と 10.0.0.0/24 をルータで接続する
- ルーター③：HSRP で VIP を用意し、デフォルトGWを冗長化する

---

## 前提

- Packet Tracer 等の検証環境を想定
- インターフェース名は g0/0/0, g0/0/1 を使用
- IP / サブネットマスク / デフォルトGW は設計通りに設定する

---

# ■ ルーター①：PC2台をルータで別ネットワークにして通信させる

---

## 1. 構成

---

### この工程でしていること
- PC1 と PC2 を別セグメント（別ネットワーク）に所属させる
- ルータを境界（L3）として通信させる

---

### 構成（IP）
```bash
PC1(192.168.1.10/24)
↓
R g0/0/0(192.168.1.1/24)
R g0/0/1(192.168.2.1/24)
↓
PC2(192.168.2.10/24)
```
---

## 2. PC 設定

---

### この工程でしていること
- PC が「同一セグメント以外」に送るときの出口（GW）を設定する

---

### 設定値

PC1
```bash
IP   : 192.168.1.10
Mask : 255.255.255.0
GW   : 192.168.1.1
```
PC2
```bash
IP   : 192.168.2.10
Mask : 255.255.255.0
GW   : 192.168.2.1
```
---

### 設定の注意点
- GW は「自分と同一セグメントにあるルータIFのIP」を指定する
- Mask が違うと同一セグメント判定が崩れ、通信できない

---

## 3. ルータ設定

---

### この工程でしていること
- ルータの各インターフェースに IP を設定し、L3 接続を成立させる
- shutdown 状態のIFを有効化する

---

### 実行コマンド
```bash
enable
conf t

interface g0/0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit

interface g0/0/1
ip address 192.168.2.1 255.255.255.0
no shutdown
exit

end
```

---

### コマンドの意味

| コマンド | 意味 |
|---|---|
| enable | 特権モード |
| conf t | 設定モード |
| interface | IF選択 |
| ip address | IP設定 |
| no shutdown | IF有効化 |
| end | 終了 |

---

## 4. 確認

---

### 確認コマンド
```bash
show ip interface brief
```
### OKの目安
- g0/0/0 と g0/0/1 が up/up

---

### 疎通確認（PC1 → PC2）
```bash
ping 192.168.2.10
```
### OKの目安
- Reply が返る

---

# ■ ルーター②：172.31.0.0/24 と 10.0.0.0/24 を接続する

---

## 1. 構成

---

### この工程でしていること
- 2つの異なるネットワークをルータで中継し、相互通信させる

---

### 構成（概要）
```bash
PC02,03(172.31.0.0/24) – SW – R – SW – PC04,05(10.0.0.0/24)
```
---

## 2. IP設計

---

### ルータ
```bash
R g0/0/0 : 172.31.0.1/24
R g0/0/1 : 10.0.0.1/24
```
### PC
```bash
PC02 : 172.31.0.2  GW 172.31.0.1
PC03 : 172.31.0.3  GW 172.31.0.1
PC04 : 10.0.0.4    GW 10.0.0.1
PC05 : 10.0.0.5    GW 10.0.0.1
```
---

### 設定の注意点
- それぞれの PC は「自分側セグメントのルータIF」を GW に設定する
- 片側だけGWが未設定だと「片方向だけ」になることがある

---

## 3. ルータ設定

---

### この工程でしていること
- 2セグメントに属する IF を作り、ルーティング可能にする

---

### 実行コマンド
```bash
enable
conf t

interface g0/0/0
ip address 172.31.0.1 255.255.255.0
no shutdown
exit

interface g0/0/1
ip address 10.0.0.1 255.255.255.0
no shutdown
exit

end
```
---

## 4. 確認

---

### ルータの経路確認
```bash
show ip route
```
### OKの目安
- C（Connected）で 172.31.0.0/24 と 10.0.0.0/24 が存在する

---

### 疎通確認（相互）
- 172.31側 → 10側
- 10側 → 172.31側

例：
```bash
ping 10.0.0.4
```
```bash
ping 172.31.0.2
```

---

# ■ ルーター③：HSRP によるデフォルトゲートウェイ冗長化

---

## 1. 要件

---

### この工程でしていること
- デフォルトGWが落ちても通信が継続できるようにする
- VIP（仮想IP）をデフォルトGWとしてPC/Serverへ設定する

---

### 要件
- PCセグメント：172.16.0.0/24
- Serverセグメント：192.168.0.0/24
- ルータ2台（R1 主系 / R2 待機系）
- VIP を各セグメントに作成し、HSRPで共有する

---

## 2. IP設計

---

### PC側（172.16.0.0/24）
```bash
VIP         : 172.16.0.254
R1 g0/0/0   : 172.16.0.1
R2 g0/0/0   : 172.16.0.2
PC          : 172.16.0.6  GW 172.16.0.254
```
### Server側（192.168.0.0/24）
```bash
VIP         : 192.168.0.254
R1 g0/0/1   : 192.168.0.1
R2 g0/0/1   : 192.168.0.2
Server      : 192.168.0.6  GW 192.168.0.254
```
---

### 設定の注意点（重要）
- PC/Server の GW は「物理ルータIP」ではなく VIP を設定する
- VIP はセグメント内で未使用の IP を選ぶ（重複NG）
- サブネットマスクの整合が取れないと VIP に到達できない

---

## 3. R1（主系）設定

---

### この工程でしていること
- R1 を Active にするため priority を高く設定する
- preempt を有効にし、復旧時に主系へ戻す

---

### 実行コマンド
```bash
enable
conf t

interface g0/0/0
ip address 172.16.0.1 255.255.255.0
standby 1 ip 172.16.0.254
standby 1 priority 120
standby 1 preempt
no shutdown
exit

interface g0/0/1
ip address 192.168.0.1 255.255.255.0
standby 2 ip 192.168.0.254
standby 2 priority 120
standby 2 preempt
no shutdown
exit

end
```

---

### コマンドの意味

| コマンド | 意味 |
|---|---|
| standby <grp> ip <vip> | HSRP VIP設定 |
| standby <grp> priority | 優先度設定（大きい方がActive） |
| standby <grp> preempt | 優先度が高ければ奪い返す |

---

## 4. R2（待機系）設定

---

### この工程でしていること
- R2 は Standby になるよう priority を低く設定する
- preempt は設計方針により付ける（今回は付与）

---

### 実行コマンド
```bash
enable
conf t

interface g0/0/0
ip address 172.16.0.2 255.255.255.0
standby 1 ip 172.16.0.254
standby 1 priority 90
standby 1 preempt
no shutdown
exit

interface g0/0/1
ip address 192.168.0.2 255.255.255.0
standby 2 ip 192.168.0.254
standby 2 priority 90
standby 2 preempt
no shutdown
exit

end
```
---

## 5. 確認（HSRP 状態）

---

### この工程でしていること
- Active / Standby が想定通りになっているか確認する

---

### 確認コマンド（ルータ）
```bash
show standby brief
```
```bash
show ip route
```
---

### OKの目安
- R1 が Active、R2 が Standby
- VIP が表示される

---

## 6. 疎通確認

---

### PC → VIP
```bash
ping 172.16.0.254
```
### PC → Server
```bash
ping 192.168.0.6
```
---

### OKの目安
- ping 成功する

---

## 7. フェイルオーバーテスト

---

### この工程でしていること
- 主系（R1）を意図的に落とし、R2 が Active へ切り替わることを確認する

---

### 実行コマンド（R1）
```bash
conf t
interface g0/0/0
shutdown
```

---

### 確認方法とOKの目安
- PC からの ping が数回 timeout 後に復旧する
- R2 が Active に切り替わる

---

### 復旧（R1）
```bash
conf t
interface g0/0/0
no shutdown
```
---

# ■ 学習ポイント

---

## ポイントまとめ

- 同一ネットワーク内の転送は L2（スイッチ）
- 異なるネットワーク間の転送は L3（ルータ）
- デフォルトGWは「自分が知らない宛先へ出る出口」
- HSRP は VIP を共有する冗長化プロトコル
- HSRP の group 番号はセグメントごとに分ける
- priority + preempt で主系制御を行う
- サブネットマスクの整合は最重要（ズレると通信不可）

---

- ルーター②：2セグメント相互通信成立
- ルーター③：GW 冗長化（フェイルオーバー）成立
